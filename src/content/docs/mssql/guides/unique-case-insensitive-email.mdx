To implement a unique and case-insensitive `email` handling in MSSQL with Drizzle, you can create a unique index on the lowercased `email` column. This way, you can ensure that the `email` is unique regardless of the case.

Drizzle has simple and flexible API, which lets you easily create such an index using SQL-like syntax:

<CodeTabs items={["schema.ts", "migration.sql"]}>
  <CodeTab>
  ```ts copy {13,14}
  import { SQL, sql } from 'drizzle-orm';
  import { AnyMsSqlColumn, int, mssqlTable, uniqueIndex, varchar } from 'drizzle-orm/mssql-core';

  export const users = mssqlTable(
    'users',
    {
      id: int().identity().primaryKey(),
      name: varchar({ length: 255 }).notNull(),
      email: varchar({ length: 255 }).notNull(),
    },
    (table) => [
      // uniqueIndex('emailUniqueIndex').on(sql`lower(${table.email})`),
      uniqueIndex('emailUniqueIndex').on(lower(table.email)),
    ]
  );

  // custom lower function
  export function lower(email: AnyMsSqlColumn): SQL {
    return sql`lower(${email})`;
  }
  ```
  </CodeTab>
  ```sql
  CREATE TABLE [users] (
	  [id] int IDENTITY(1, 1),
	  [name] varchar(255) NOT NULL,
	  [email] varchar(255) NOT NULL,
	  CONSTRAINT [users_pkey] PRIMARY KEY([id])
  );
  --> statement-breakpoint
  CREATE UNIQUE INDEX [emailUniqueIndex] ON [users] (lower([email]));
  ```
</CodeTabs>

<Callout>
In SQL Server, string comparison case-sensitivity is determined by the column's
collation. On a case-insensitive collation (the common default, e.g. `SQL_Latin1_General_CP1_CI_AS`)
comparisons already ignore case; the unique index on `lower(email)` guarantees
uniqueness regardless of case even when a case-sensitive collation is used.
</Callout>

This is how you can select user by `email` with `lower` function:

<Section>
```ts copy {10}
import { eq } from 'drizzle-orm';
import { lower, users } from './schema';

const db = drizzle(...);

const findUserByEmail = async (email: string) => {
  return await db
    .select()
    .from(users)
    .where(eq(lower(users.email), email.toLowerCase()));
};
```

```sql
select * from [users] where lower([users].[email]) = 'john@email.com'
```
</Section>
