MSSQL doesn't have an array or a native `json` data type, but you can store a JSON array in an `nvarchar` column and work with it using SQL Server's built-in JSON functions. To set an empty array as a default value you can use the `sql` operator with `'[]'` syntax:

<Section>
```ts copy {9}
import { sql } from 'drizzle-orm';
import { int, mssqlTable, nvarchar } from 'drizzle-orm/mssql-core';

export const users = mssqlTable('users', {
  id: int().identity().primaryKey(),
  tags: nvarchar({ length: 'max', mode: 'json' })
    .$type<string[]>()
    .notNull()
    .default(sql`'[]'`),
});
```

```sql
CREATE TABLE [users] (
	[id] int IDENTITY(1, 1),
	[tags] nvarchar(max) NOT NULL CONSTRAINT [users_tags_default] DEFAULT ('[]'),
	CONSTRAINT [users_pkey] PRIMARY KEY([id])
);
```
</Section>

The `mode: 'json'` option defines how values are handled in the application. With `json` mode, values are treated as JSON object literal.

You can specify `.$type<..>()` for json object inference, it will not check runtime values. It provides compile time protection for default values, insert and select schemas.
