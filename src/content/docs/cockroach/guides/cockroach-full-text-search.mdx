This guide demonstrates how to implement full-text search in CockroachDB with Drizzle ORM. Full-text search is a technique used to search for text within a document or a set of documents. CockroachDB provides a set of functions to work with full-text search, such as `to_tsvector` and `to_tsquery`.

The `to_tsvector` function parses a textual document into tokens, reduces the tokens to lexemes, and returns a `tsvector` which lists the lexemes together with their positions in the document:

<Section>
```ts copy {6}
import { sql } from 'drizzle-orm';

const db = drizzle(...);

await db.execute(
  sql`select to_tsvector('english', 'Guide to CockroachDB full-text search with Drizzle ORM')`,
);
```
</Section>

The `to_tsquery` function converts a keyword to normalized tokens and returns a `tsquery` that matches the lexemes in a `tsvector`. The `@@` operator is used for direct matches:

<Section>
```ts copy {2, 3}
await db.execute(
  sql`select to_tsvector('english', 'Guide to CockroachDB full-text search with Drizzle ORM')
    @@ to_tsquery('english', 'Drizzle') as match`,
);
```

```json
[ { match: true } ]
```
</Section>

Drizzle doesn't support the `tsvector` type natively, so you need to convert your data in the `text` column on the fly. To enhance the performance, you can create a `GIN` index on your column like this:

<CodeTabs items={["schema.ts", "migration.sql"]}>
  <CodeTab>
  ```ts copy {10, 11}
  import { sql } from 'drizzle-orm';
  import { cockroachTable, index, int4, text } from 'drizzle-orm/cockroach-core';

  export const posts = cockroachTable(
    'posts',
    {
      id: int4().primaryKey().generatedAlwaysAsIdentity(),
      title: text().notNull(),
    },
    (table) => [
      index('title_search_index').using('gin', sql`to_tsvector('english', ${table.title})`),
    ]
  );
  ```
  </CodeTab>
  ```sql
  CREATE TABLE "posts" (
          "id" int4 PRIMARY KEY GENERATED ALWAYS AS IDENTITY (INCREMENT BY 1 MINVALUE 1 MAXVALUE 2147483647 START WITH 1 CACHE 1),
          "title" string NOT NULL
  );
  --> statement-breakpoint
  CREATE INDEX "title_search_index" ON "posts"
    USING gin (to_tsvector('english', "title"));
  ```
</CodeTabs>

To implement full-text search you can use the `to_tsvector` and `to_tsquery` functions with `sql` operator:

<Section>
```ts copy {9}
import { sql } from 'drizzle-orm';
import { posts } from './schema';

const title = 'trip';

await db
  .select()
  .from(posts)
  .where(sql`to_tsvector('english', ${posts.title}) @@ to_tsquery('english', ${title})`);
```

```sql
select * from "posts"
  where to_tsvector('english', "title") @@ to_tsquery('english', 'trip');
```
</Section>

To match by any of the keywords, you can use the `|` operator:

<Section>
```ts copy {6}
const title = 'Europe | Asia';

await db
  .select()
  .from(posts)
  .where(sql`to_tsvector('english', ${posts.title}) @@ to_tsquery('english', ${title})`);
```
</Section>

To match multiple keywords, you can use the `plainto_tsquery` function:

<Section>
```ts copy {7}
// 'discover & Italy'
const title = 'discover Italy';

await db
  .select()
  .from(posts)
  .where(sql`to_tsvector('english', ${posts.title}) @@ plainto_tsquery('english', ${title})`);
```

```sql
select * from "posts"
  where to_tsvector('english', "title") @@ plainto_tsquery('english', 'discover Italy');
```
</Section>

To match a phrase, you can use the `phraseto_tsquery` function:

<Section>
```ts copy {8}
// if you query by "trip family", it will not return any result
// 'family <-> trip'
const title = 'family trip';

await db
  .select()
  .from(posts)
  .where(sql`to_tsvector('english', ${posts.title}) @@ phraseto_tsquery('english', ${title})`);
```

```sql
select * from "posts"
  where to_tsvector('english', "title") @@ phraseto_tsquery('english', 'family trip');
```
</Section>

To rank the search results, you can use the `ts_rank` function and `orderBy` method:

<Section>
```ts copy {5,9,10,13}
import { desc, getColumns, sql } from 'drizzle-orm';

const search = 'trip';

const matchQuery = sql`to_tsvector('english', ${posts.title}), to_tsquery('english', ${search})`;

await db
  .select({
    ...getColumns(posts),
    rank: sql`ts_rank(${matchQuery})`,
  })
  .from(posts)
  .where(sql`to_tsvector('english', ${posts.title}) @@ to_tsquery('english', ${search})`)
  .orderBy((t) => desc(t.rank));
```

```sql
select *, ts_rank(to_tsvector('english', "title"), to_tsquery('english', 'trip')) as "rank"
from "posts"
where to_tsvector('english', "title") @@ to_tsquery('english', 'trip')
order by "rank" desc;
```
</Section>

<Callout type="warning">
The following PostgreSQL full-text search features are **not supported by CockroachDB** and
will fail at runtime:

- `websearch_to_tsquery()` — not implemented.
- `setweight()` and concatenating `tsvector`s with the `||` operator — not implemented, so
  multi-column weighted search (searching across `title` + `description` with different weights)
  does not work. Search one `to_tsvector` column expression at a time instead.
</Callout>
